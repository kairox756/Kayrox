Objetivo del proyecto

TermMik es una consola modular desarrollada por Kayrox, basada en Mikota Core. Su propósito es ofrecer un entorno técnico seguro y extensible para desarrolladores, con funciones como liberación validada, verificación de integridad, logs, modo seguro y soporte OTA.

Análisis del código MikCode

El fragmento anterior define la estructura del sistema Mikota Core y su integración con TermMik:

  • info mikCore(): Define el núcleo del sistema, versión, arquitectura y compatibilidad.
  • info mikShell(): Describe la capa de ejecución (Shell) con versión, codename y dependencias Python.
  • def system: Especifica el entorno de desarrollo, lenguaje base y plataformas compatibles.
  • aplicacion_console_for_windows(): Define la versión del programa y su tipo de aplicación (consola).

Arquitectura del sistema

TermMik se compone de módulos independientes:

  • Mikota Core: Motor principal que gestiona compatibilidad, rendimiento y seguridad.
  • Release System: Valida códigos de liberación mediante JSON remoto.
  • Integrity Module: Verifica hashes SHA256 antes de aplicar actualizaciones.
  • Safe Mode: Permite recuperación de archivos dañados desde el servidor OTA.
  • Logs: Registra cada acción ejecutada por el usuario.
  • Plugins: Extiende funciones sin modificar el núcleo.

Tecnologías propias

TermMik utiliza tecnologías desarrolladas internamente por Kayrox:

  • Mikota Core: Framework interno que actúa como motor de compatibilidad y seguridad.
  • MikCode (.mkay): Lenguaje DSL propio para definir metadatos y configuraciones del sistema.
  • Kayrox OTA: Sistema de actualización remota con validación de integridad.
  • TermMik Shell: Consola Python personalizada con comandos modulares.
